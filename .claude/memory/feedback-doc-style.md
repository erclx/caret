---
name: Doc style
description: Match existing depth and tone when editing any project doc
type: feedback
---

When adding to any `.claude/` or `standards/` doc, read surrounding content first and match its depth, length, and tone. State the decision and reason, then stop.

**Why:** Entries that are too verbose or detailed need rewriting every time.

**How to apply:** Calibrate to the adjacent content before writing.
