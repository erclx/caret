---
name: Task entry style
description: How to write TASKS.md entries at a high level, not implementation-specific
type: feedback
---

Keep task entries high level. Describe what and why, not how. Match the style of existing done entries in TASKS.md: short bullet per item, no code references, no class names or prop details.

**Why:** User corrected over-specific entries during a session. PLANNER.md also enforces this rule explicitly.

**How to apply:** Before writing a task entry, check an existing done entry for tone. If you're tempted to mention a class name, file line, or implementation detail, cut it.
